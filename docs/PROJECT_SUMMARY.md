# 프로젝트 요약

## 목적

자영업자가 스마트폰으로 찍은 음식 사진을 배달앱·SNS·광고에 바로 쓸 수 있는 수준으로 변환한다.

사진 촬영 전문 지식이 없어도, 스튜디오가 없어도, 1장의 원본 사진만 있으면 된다.

## 핵심 개념

**템플릿 = 분위기**

템플릿은 특정 음식 종류가 아니라 촬영 스타일을 정의한다. 대표 이미지는 "이 분위기"가 어떻게 보이는지 설명하는 예시일 뿐이다. 사용자는 어떤 음식이든 업로드할 수 있고, 선택한 템플릿 규칙이 그 음식에 적용된다.

예: `dark-premium` 템플릿을 선택하면 후라이드 치킨도, 삼겹살도, 떡볶이도 같은 "어두운 바 분위기"로 변환된다.

## 사용자 흐름 (3단계)

```
[1단계] 사진 업로드
         ↓
[2단계] 스타일 선택
         · 16개 템플릿 중 하나 선택
         · 카테고리 입력 (선택사항 — 질감 보존 정밀도 향상)
         · delivery-hero 템플릿은 추가로 음식 보존 강도(strict/natural/abundant) 선택
         ↓
[3단계] AI 생성 → A/B 2장 동시 출력
         · A: 기본 스튜디오형 (음식 중심, 소품 0~1개)
         · B: 프리미엄 소품형 (소품·가니쉬·공간 연출 강화)
         ↓
[결과] 마음에 드는 시안 선택 → PNG 다운로드
```

## A/B 출력 구조

두 시안은 같은 템플릿 촬영 규칙(배경·조명·색온도·무드)을 공유한다. 차이는 소품 밀도와 음식 점유율에서 나온다.

| | 시안 A (기본 스튜디오형) | 시안 B (프리미엄 소품형) |
|---|---|---|
| 음식 점유율 | `foodRatioA` (더 높음) | `foodRatioB` (더 낮음 — 공간 확보) |
| 필수 용기·표면 | 첫 번째 1개만 | 전부 |
| 음식 관련 소품 | 없음 | 최대 2개 |
| 공간 연출 소품 | 없음 | 3~5개 (템플릿 광고 컨셉의 핵심) |
| 가니쉬 | 음식 위에 올라가는 것만 | 전부 |

## 음식 보존 강도 (delivery-hero 전용)

`strict / natural / abundant` 3단계. `delivery-hero` 외 모든 템플릿은 `strict` 고정이다.

| 모드 | 음식 변화 범위 |
|---|---|
| strict | 조명·색 정리만. 형태·양·배치 완전 고정. |
| natural | 색감·윤기 자연 보정 허용. 형태·양 고정. |
| abundant | 광고급 광택·볼륨감 강조 허용. 메뉴 정체성 유지. |

## 기술 스택

| 레이어 | 기술 |
|---|---|
| 프레임워크 | Next.js (App Router) |
| 이미지 생성 (GPT) | OpenAI `gpt-image-1` — `images.edit` API |
| 이미지 생성 (Flux) | fal.ai `fal-ai/flux-pro/kontext` |
| 배경 제거 | Photoroom API (개발 테스트용) |
| 스타일링 | Tailwind CSS |

## API 라우트

| 경로 | 역할 |
|---|---|
| `/api/generate-photo` | GPT — A/B 2회 병렬 호출 |
| `/api/generate-flux` | Flux — A/B 2회 병렬 호출 |
| `/api/generate-preserve` | GPT — 마스크 기반 편집 (실험) |
| `/api/generate-composite` | GPT — 원본 픽셀 합성 (실험) |
| `/api/photoroom-remove-bg` | Photoroom 배경 제거 (개발 전용) |
| `/api/photoroom-compose-test` | Photoroom 배경 합성 (개발 전용) |
| `/api/ai-background-compose` | AI 배경 생성 + Photoroom 합성 (실험) |

## 현재 상태 (2026-06-21)

- 1차 MVP: 16개 템플릿, A/B 2시안 생성, PNG 다운로드 완성
